# AI 语音绘图工具 — 设计文档

> **比赛题目：** 题目二 — AI 语音绘图工具（纯语音控制，无鼠标键盘）
> **完成时限：** 24 小时
> **文档版本：** v1.0
> **编写日期：** 2026-06-12

---

## 项目简介

### 一句话概述

**VoiceDraw** 是一款运行在浏览器中的纯语音控制绘图应用。用户无需触碰鼠标或键盘，仅通过自然语言语音指令即可完成从简单图形、自由笔触到多对象组合的完整绘图创作。应用将所有操作实时反馈于画布，并通过语音合成与用户进行双向自然对话。

### 背景与动机

传统绘图工具（如 Photoshop、Illustrator、画图）高度依赖精确的鼠标/键盘操作，这在以下场景存在天然痛点：

| 场景 | 痛点 |
|------|------|
| **无障碍需求** | 运动障碍用户无法使用键鼠，但具备语言表达能力 |
| **远距离操作** | 会议投屏时，演讲者远离电脑却需要在白板上标注 |
| **儿童创作** | 低龄用户手指精细控制弱，但能用话语描述想画的内容 |
| **双手被占用** | 厨师、工人、实验员等需要记录视觉信息但手不干净 |
| **快速原型** | 头脑风暴时口头说出比拖拽搭建更符合思维节奏 |

语音交互+生成式绘图的组合，在 2025-2026 年因 Web Speech API 成熟度和 Canvas 引擎的完善，已具备落地条件。VoiceDraw 正是瞄准这一交汇点。

### 核心设计挑战

比赛题目中明确提出四个维度的考察重点，我们的应对思路如下：

| 考察维度 | 核心挑战 | 设计应对 |
|----------|----------|----------|
| **指令理解的准确性** | 中文口语的随意性、同音字、方言口音 | 多层规则引擎：精确匹配 → 模糊匹配 → 正则提取 → 容错兜底 |
| **指令理解的容错性** | 识别错误时不能崩溃，需优雅降级 | 同音字映射表 + 编辑距离容忍 + "未听清请重复"反馈闭环 |
| **语音到绘图的响应延迟** | 用户说完到画面变化的心理预期 < 500ms | 纯浏览器端处理（零网络往返）+ 指令解析 < 10ms + Canvas 操作 < 16ms |
| **复杂指令的拆解与执行** | "画一个大的红色圆在中间" 需一次解析多参数 | 结构化意图提取 + 位置上下文记忆 + 连续指令标点拆分 |

### 设计目标

1. **零依赖运行** — 打开浏览器即用，无需安装任何软件或申请 API Key
2. **真正的无触控** — 从启动应用到导出图片的完整链条，手不需要碰鼠标键盘
3. **直觉式对话** — 指令用自然口语而非死板的命令格式，支持"画个红色的圆"而非"circle red radius 50"
4. **实时反馈** — 每个操作有视觉结果和语音确认双重反馈
5. **24 小时可落地** — 技术选型服务于比赛时间约束，重框架复用、轻自研轮子

### 目标用户画像

| 角色 | 使用方式 |
|------|----------|
| **评 委** | 打开浏览器，说"画一个蓝色的三角形"，验证画布立即出现对应图形 |
| **无障碍用户** | 全程通过语音完成从构思到导出的完整绘图流程 |
| **普通创作者** | 当手不方便或想快速草图时，语音作为最高效的输入方式 |

## 1. 技术选型

### 1.1 总体方案：纯前端 SPA + 浏览器原生语音 API

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| **框架** | React 18 + TypeScript | 组件化开发效率高，类型安全，社区生态最强 |
| **构建工具** | Vite 5 | 秒级 HMR，构建速度快，零配置开箱即用 |
| **画布引擎** | Fabric.js 6.x | 内置自由绘制模式、图形创建/选择/变换、文本编辑、SVG 导出、序列化，减少 60%+ 画布样板代码 |
| **语音识别** | Web Speech API (`SpeechRecognition`) | 浏览器原生支持 Chrome/Edge，无需后端、无需 API Key、零延迟部署 |
| **语音合成** | Web Speech API (`SpeechSynthesis`) | 浏览器原生 TTS，用于语音反馈确认 |
| **指令解析** | 自研正则+关键词匹配引擎 | 轻量级，无外部依赖，毫秒级解析 |
| **样式方案** | Tailwind CSS 4.x | 原子化 CSS，快速搭建 UI |
| **状态管理** | React Context + useReducer | 轻量足够，不需引入 Redux/Zustand |
| **测试** | Vitest + Playwright | 单元测试 + E2E 视觉回归测试 |

### 1.2 为什么选 Web Speech API 而非云端方案

| 维度 | Web Speech API | OpenAI Whisper API | 阿里云/讯飞 |
|------|:---:|:---:|:---:|
| **部署复杂度** | ⭐ 零配置 | 🔴 需服务端中转 | 🔴 需服务端中转 |
| **延迟** | ⭐ 200-300ms | 🟡 500-1500ms | 🟡 400-1000ms |
| **费用** | ⭐ 完全免费 | 🔴 按 token 计费 | 🔴 按调用计费 |
| **中文识别率** | ⭐ 92-95%（安静环境） | ⭐ 95-98% | ⭐ 94-97% |
| **离线能力** | 🟡 Edge 支持本地识别 | ❌ 必须联网 | ❌ 必须联网 |
| **HTTPS 要求** | 🟡 需要 localhost 或 HTTPS | 🔴 需要服务端 | 🔴 需要服务端 |
| **24h 实现难度** | ⭐ 极低（纯前端） | 🔴 高（需后端） | 🔴 高（需后端） |

**结论：Web Speech API 是 24 小时比赛中唯一可行的方案。**

### 1.3 为什么选 Fabric.js 而非其他 Canvas 库

| 维度 | Fabric.js | Konva.js | 原生 Canvas |
|------|:---:|:---:|:---:|
| **自由绘制** | ⭐ 内置 `PencilBrush` | 🟡 需手动实现 | 🔴 全部手写 |
| **图形创建** | ⭐ 内置 Rect/Circle/Triangle/Line 等 | ⭐ 内置基础形状 | 🔴 全部手写 |
| **对象选择/变换** | ⭐ 开箱即用 | ⭐ 开箱即用 | 🔴 全部手写 |
| **文本编辑** | ⭐ 画布上直接编辑 | 🟡 基础支持 | 🔴 全部手写 |
| **撤销/重做** | 🟡 需手动实现 | 🟡 需手动实现 | 🔴 全部手写 |
| **序列化** | ⭐ canvas.toJSON() | ⭐ stage.toJSON() | 🔴 全部手写 |
| **24h 开发效率** | ⭐ 极高 | ⭐ 高 | 🔴 极低 |

**结论：Fabric.js 是为绘图编辑器场景预制的，24 小时内能实现的功能最多。**

### 2.2 核心数据流

```
用户语音 ──▶ SpeechRecognition ──▶ 文本转录
                                       │
                                       ▼
                                 CommandParser
                              (意图识别 + 参数提取)
                                       │
                              ┌────────┼────────┐
                              ▼        ▼        ▼
                          ShapeCmd  DrawCmd  EditCmd
                              │        │        │
                              ▼        ▼        ▼
                         CanvasEngine (Fabric.js)
                              │
                              ▼
                         Canvas 更新 + TTS 反馈
```

### 2.3 模块职责

| 模块 | 文件 | 职责 |
|------|------|------|
| **VoiceInput** | `src/voice/VoiceRecognizer.ts` | 封装 SpeechRecognition，管理 continuous/interim 模式，处理各类语音错误 |
| **CommandParser** | `src/parser/CommandParser.ts` | 正则+关键词匹配，从转录文本中提取指令意图和参数 |
| **CanvasEngine** | `src/canvas/CanvasEngine.ts` | 封装 Fabric.js，提供统一的绘制接口（形状、自由线、文本、编辑操作） |
| **StateManager** | `src/state/DrawingState.ts` | 管理当前工具、颜色、笔刷大小、选中对象、撤销栈 |
| **VoiceFeedback** | `src/voice/VoiceFeedback.ts` | 封装 SpeechSynthesis，指令确认/错误提示 |
| **AudioVisualizer** | `src/components/AudioVisualizer.tsx` | 麦克风状态波形动画 |
| **ToolBar** | `src/components/ToolBar.tsx` | 当前状态指示器（仅显示，不可交互） |
| **Canvas** | `src/components/DrawingCanvas.tsx` | Fabric.js 画布容器 |
| **CommandLog** | `src/components/CommandLog.tsx` | 语音识别历史 / 指令执行日志 |

### 2.4 指令解析引擎设计

```
输入文本: "画一个红色的圆在画布中间"
           │
           ▼
┌─────────────────────┐
│  1. 分词 & 标准化    │  → ["画", "一个", "红色", "的", "圆", "在", "画布", "中间"]
└─────────────────────┘
           │
           ▼
┌─────────────────────┐
│  2. 意图识别         │  → 匹配 "画" → INTENT_DRAW_SHAPE
└─────────────────────┘
           │
           ▼
┌─────────────────────┐
│  3. 参数提取         │  → shape: "circle", color: "red", position: "center"
└─────────────────────┘
           │
           ▼
┌─────────────────────┐
│  4. 指令验证         │  → 检查必填参数完整性
└─────────────────────┘
           │
           ▼
┌─────────────────────┐
│  5. 容错处理         │  → 模糊匹配 ("园"→"圆", "举行"→"矩形")
└─────────────────────┘
```

**意图分类体系：**

```
INTENT_DRAW_SHAPE     — 绘制指定形状
INTENT_FREEHAND       — 开始/停止自由绘制
INTENT_CHANGE_COLOR   — 更改颜色
INTENT_CHANGE_SIZE    — 更改笔刷/形状尺寸
INTENT_MOVE           — 移动对象
INTENT_DELETE         — 删除对象
INTENT_UNDO           — 撤销
INTENT_REDO           — 重做
INTENT_CLEAR          — 清空画布
INTENT_SAVE           — 导出/保存
INTENT_FILL           — 填充
INTENT_STROKE         — 描边
INTENT_TEXT           — 插入文本
INTENT_ZOOM           — 缩放画布
INTENT_SELECT         — 选择对象
INTENT_DESELECT       — 取消选择
INTENT_UNKNOWN        — 无法识别
```

## 4. 功能优先级排序

## 附录 A：语音指令容错策略

### A.1 同音字/谐音容错

```
"园" → "圆"     "举行" → "矩形"    "方形" → "矩形"
"删除" ← "山出"  "回退" ← "灰退"    "撤销" ← "车销"
"花" → "画"     "巨型" → "矩形"    "王" → "黄"
```

### A.2 意图匹配优先级

1. 精确匹配（直接命中关键词）
2. 模糊匹配（编辑距离 ≤1 的中文词）
3. 正则提取（从自然语言中匹配意图模式）
4. 语义兜底（所有匹配失败 → "未识别指令"语音提示）

### A.3 连续识别下的噪声过滤

- 句长 < 1 字 → 忽略
- 句长 > 50 字 → 取前 50 字
- 无任何意图关键词 → 忽略（不提示，避免烦扰）
- 相同指令 500ms 内重复 → 去重
